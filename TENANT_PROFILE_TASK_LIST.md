# 租户下人员画像改造任务清单

## 背景

当前系统里的“用户管理”更接近“控制台账号 + 人员画像”的混合概念，但实际业务模型应当是：

- 租户：购买系统的公司
- 人员画像：租户下真实使用机器人的人员，不登录后台
- 后台账号：平台侧管理员、租户管理员等控制台操作者

本次改造目标是将“用户管理”重定义为“租户下的人员画像管理”，并将后台账号与业务画像彻底分离。

## 目标

- 将“用户管理”页面调整为“人员画像”页面
- 页面以租户为一级筛选维度
- 业务侧人员不再按后台登录账号建模
- 后端查询、导出、详情接口按租户范围返回画像数据
- 保留现有标签、备注、语言偏好、来源渠道、平台账号绑定等画像能力

## 核心原则

- 同名人员跨租户不能合并
- 同一外部账号跨租户不能合并
- 控制台账号与业务画像分离
- 平台管理员可跨租户查看
- 租户管理员默认只看本租户
- 真实业务用户不登录后台，只通过渠道接入产生画像
