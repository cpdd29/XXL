# Wiki Schema

本文件定义 XXL wiki 的编写和维护规则。

## 页面类型

- `project/`: 项目稳定事实、架构边界、模块入口
- `ops/`: 启动、部署、验收、排障
- `faq/`: 高频问题和标准回答口径

## 编写规则

- 一页只讲一个稳定主题。
- 先写结论，再写入口、边界、例外。
- 优先写“去哪里看、怎么判断、什么时候更新”，不要复制整段原始长文档。
- wiki 与代码或真源文档不一致时，以真源为准，并在同一次改动里更新 wiki。
- 页面内容应尽量稳定，避免写临时状态、一次性日志和短期结论。

## 推荐结构

```md
# 页面标题

## 结论

一句话说明这个主题在 XXL 里的准确定义。

## 入口

- 代码入口
- 文档入口
- API 或脚本入口

## 边界

- 什么属于这个主题
- 什么不属于这个主题

## 真源

- 相关代码或文档路径
```

## 更新时机

- 新模块正式落仓
- 主链执行路径变化
- 启动/部署/验收流程变化
- 常见问答重复出现，已经形成稳定答案
